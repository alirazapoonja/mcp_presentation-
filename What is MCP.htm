What is a REST API?

A REST API is a way for two computer systems to communicate over the internet using standard HTTP methods.

 

It allows applications (websites, mobile apps, servers, AI tools, etc.) to send requests and receive data from another system.

 

Think of a REST API as a waiter in a restaurant 🍽️:

  • Client (You) → The person ordering food
  • API (Waiter) → Takes the request to the kitchen
  • Server (Kitchen) → Prepares the food
  • Response (Food served) → The result returned to you

 

1️⃣ Endpoint

An endpoint is a URL where the API receives requests.

2️⃣ HTTP Methods (Actions)

REST APIs use standard HTTP methods:

Method

      Purpose

Example

GET

   Retrieve data

 Get list of products

POST

   Create new data

Add a new product

PUT

   Update data

Update product details

DELETE

   Remove data

Delete a product

 

3️⃣ Request

The client sends a request to the server

 

4️⃣ Response

The server sends data back, usually in JSON format.

{
"id": 10,
"name": "Laptop",
"price": 750,
"stock": 12
}

MCP Tools (Model-Controlled Primitive)

Tools are executable functions exposed by an MCP server that allow AI models to perform actions such as creating, updating, or deleting data in external systems.

They follow a model-controlled paradigm — the AI model autonomously discovers available tools via tools/list and decides when to invoke them via tools/call.

 

MCP (Model Context Protocol) Tools wo external functions ya services hoti hain jo AI ko real duniya ke kaam karne ki power deti hain.

MCP Resources (App-Controlled Primitive)

 

Resources are read-only data endpoints exposed by an MCP server, allowing AI models to access pre-authorized information such as documents, configurations, and reference data.

 

They follow an app-controlled paradigm — the host application decides which data to expose, and the model simply reads what is made available via resources/list and resources/read.

MCP Prompts (User-Controlled Primitive)

They follow a user-controlled paradigm — the user explicitly selects which prompt to apply (e.g., via slash commands like /contract_review), unlike tools where the model decides autonomously.

JSON-RPC — The Language of MCP?

What is JSON-RPC?

Think of your MCP Client as a person walking into a shop (MCP Server) to ask for something. The way they ask and receive — that specific language — is JSON-RPC.

1 Request

The client asks the server to do something

2 Response

The server replies back.

3 Error

If something goes wrong:

4 Notification (Bonus)

A notification has no id. The client just informs the server — no reply needed.

5 Simple Analogy

JSON-RPC

Real Life

Request

Placing a pizza order

id

Your order/token number

method

Which pizza you want

result

Pizza delivered

error

Pizza not available

notification

"I'm on my way" — no reply expected

 

 

JSON-RPC ek lightweight communication protocol hai jo client aur server ke darmiyan baat karne ka tareeqa define karta hai — using JSON format.

 

“JSON-RPC = Message format jisme AI Agent MCP server se baat karta hai”

 

Biryani Style Samajh Lo

  •  Rice = JSON (data format)
  •  Chicken = RPC (Remote Procedure Call)
  •  Masala = Method + Params
  •  Dum = Response

Result:
Perfect Biryani = JSON + RPC = JSON-RPC

{  "jsonrpc": "2.0",   "result": " Biryani Ready",   "id": 1}

MCP Transports

A transport is the method used to send messages between the Client and Server. Think of it as the road between two houses — the houses are the same, but the road can be different.

MCP Transports ka matlab hai woh raaste (channels) jahan se Client aur Server ke darmiyan data travel karta hai in Model Context Protocol (MCP).

Transport = Data bhejne aur receive karne ka tareeqa

Common MCP Transports:

  • HTTP (REST style)
  • WebSocket (real-time)
  • STDIO (local communication)
  • SSE (Server-Sent Events)

Socho tum biryani bana rahe ho

  • Chef (AI Agent) → order deta hai
  • Waiter (Client) → message le kar jata hai
  • Kitchen (Server) → biryani banata hai
  • Transport (raasta) → waiter ka chalne ka tareeqa

Agar waiter:

  • bike pe jaye = HTTP 🚴
  • walk kare = STDIO 🚶
  • live call kare = WebSocket 📞

Toh yeh sab MCP Transports hain

Why We Use MCP Transports?

Hum MCP Transports use karte hain kyunki:

  1. Communication enable hota hai
    • Client aur Server baat kar sakte hain
  2. Different environments support
    • Local machine → STDIO
    • Internet → HTTP/WebSocket
  3. Speed & efficiency control
    • Real-time → WebSocket
    • Simple request → HTTP

How We Use MCP Transports?

Step-by-step flow:

  1. AI Agent request banata hai
  2. Client us request ko transport ke through bhejta hai
  3. Server request process karta hai
  4. Response wapas same transport se aata hai

Example:

  • Client → HTTP request bhejta hai
  • Server → JSON-RPC response deta hai
  • Result → AI ko mil jata hai

Types of MCP Transports

       3. STDIO (Local ninja)

SSE (Server-Sent Events)

SSE means the server runs independently on a port (like a website) and clients connect to it over HTTP. The server stays alive and keeps an open connection so it can push messages back to the client in real-time.

 

Socho tum ek biryani hotel chala rahe ho
Aur customer (browser/app) table par baitha hai.

 

Server (chef) khud se continuously updates bhejta rehta hai client ko (customer ko) — bina baar baar request ke.

 

Server-Sent Events ek technology hai jahan server continuously data push karta hai client ko ek open connection par.

 

2. Why we use SSE?

Socho agar har dafa customer waiter ko bula kar bole:
“Bhai biryani ready hui?”

Ye inefficient hai!

SSE solve karta hai:

  • Bar bar request ki zaroorat nahi
  • Real-time updates milte hain
  • Lightweight hai (WebSocket se simple)

Use cases:

  • Live notifications
  • Stock updates
  • Chat (simple one-way)
  • AI streaming responses (important!)

 

3. How we use SSE?

·       Client connection open karta hai:

·       Server stream start karta hai:

·       Client continuously sunta rehta hai:

 

4. Why SSE is Important in AI?

·       AI apps (jaise ChatGPT type systems) mein:

·       AI apps (jaise ChatGPT type systems) mein:

·       Response ek dum nahi aata
Token by token stream hota hai

SSE ka role:

  • Fast streaming responses
  • Real-time AI output
  • Low complexity vs WebSockets
